3. **Tier 3 — Fallback**: if neither Tier 1 nor Tier 2 is available, direct the customer to the fallback URL or human handoff channel.

- This agent does NOT proxy or man-in-the-middle routed operations

## External Agent Interaction Protocol

This contract is intended to be readable by external agents (user agents, platform agents) that discover this brand agent via its `agent-card.json`.

**How to call this brand agent**:

1. Discover via ACN or direct A2A agent card lookup
2. Read `agent-card.json` for declared capabilities and endpoint
3. Send requests using A2A protocol v0.3.0
4. For high-risk operations (marked "Requires confirmation" above), always present the operation to the end user before invoking

**Error handling**:

| Situation | Expected agent behavior |
|---|---|
| Service data unavailable | Acknowledge honestly, do not fabricate. Offer to retry or redirect. |
| Operation fails | Report the failure clearly, suggest the most likely next action. |
| Partial success | Report what succeeded and what did not. Do not claim full success. |
| User input invalid | Explain what was wrong and what valid input looks like. |
| Request is out of scope | Acknowledge the gap, direct to appropriate channel. |
| Request hits a hard limit | Decline clearly without being dismissive. Explain what is possible instead. |
